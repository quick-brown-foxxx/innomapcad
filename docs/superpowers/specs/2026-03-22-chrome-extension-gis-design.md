# Design: Chrome Extension ГИС-САПР для цифрового двойника Иннополиса

> **Дата**: 2026-03-22
> **Статус**: Draft
> **Контекст**: хакатон, кейс ОЭЗ «Иннополис», сокращённый скоуп

---

## 1. Концепция

Chrome Extension инжектируется в цифровой двойник Иннополиса (4dinno.ru/map/territory) и добавляет поверх него ГИС-САПР возможности: слои ограничений и интерактивное размещение объектов с валидацией.

**Демо-сценарий:** проектировщик открывает двойник → включает слои кадастра и охранных зон → выбирает пресет здания → ставит на карту → видит конфликт → двигает → зелёный статус.

## 2. Архитектура

Два компонента:

### 2.1 Chrome Extension (content script)

- Инжектируется в `4dinno.ru/map/*`
- Получает доступ к deck.gl через `window.deck`
- Добавляет свои слои и плавающую UI-панель поверх карты
- Turf.js для мгновенного визуального фидбека (пересечения, буферы)

### 2.2 Лёгкий бэкенд (FastAPI + Shapely)

- Один контейнер, данные в памяти из GeoJSON-файлов
- Финальная серверная валидация размещения
- Отдача слоёв и каталога пресетов

### 2.3 Данные

Реальные данные Иннополиса выгружаются отдельным подготовительным этапом и захардкоживаются как GeoJSON-файлы в репозитории. Источники: НСПД (кадастр, ЗОУИТ).

## 3. Chrome Extension — детали

### 3.1 UI — плавающая панель

Панель поверх карты (не в сайдбаре двойника), содержит:

- **Переключатели слоёв** — два тогла: «Кадастровые участки», «Охранные зоны»
- **Палитра объектов** — 5 пресетов (карточки с иконками)
- **Панель валидации** — статус размещения и список конфликтов

### 3.2 deck.gl слои

| Слой | Тип | Назначение |
|---|---|---|
| Кадастровые участки | GeoJsonLayer | Полупрозрачная заливка + обводка |
| Охранные зоны | GeoJsonLayer | Красная полупрозрачная заливка |
| Размещаемый объект | SolidPolygonLayer | Зелёный/красный по статусу валидации |
| Буфер отступа | GeoJsonLayer | Пунктирная обводка вокруг объекта |

### 3.3 Взаимодействие при размещении

1. Выбор пресета → клик по карте → объект появляется
2. Turf.js мгновенно проверяет `intersect` с зонами → цвет объекта
3. Drag для перемещения → пересчёт на каждый move
4. Кнопка «Проверить» → `POST /validate` на бэкенд → финальный результат

### 3.4 Интеграция с двойником

Изолированная: только добавление своих deck.gl слоёв и UI. Не трогаем существующие слои, контролы, сайдбар.

**Stretch goal:** перехват кликов по объектам двойника через deck.gl picking → расширенная карточка с кадастровыми данными.

## 4. Бэкенд API

### 4.1 Эндпоинты

| Метод | Путь | Назначение |
|---|---|---|
| `POST /api/v1/validate` | Валидация размещения | Геометрия + пресет → список конфликтов |
| `GET /api/v1/layers/{slug}` | Отдача GeoJSON слоёв | cadastral, protection_zones |
| `GET /api/v1/presets` | Каталог пресетов | Типы объектов с параметрами |
| `GET /health` | Healthcheck | Для Docker |

### 4.2 Валидация — запрос/ответ

```json
// POST /api/v1/validate
// Request:
{ "geometry": { "type": "Polygon", "coordinates": [...] }, "preset_slug": "residential" }

// Response:
{ "valid": false, "conflicts": [
  { "layer": "protection_zones", "type": "intersection", "description": "Пересечение с охранной зоной газопровода" }
]}
```

### 4.3 Типы проверок (Shapely)

- `intersects` — пересечение с охранными зонами
- `within` — объект внутри границ кадастрового участка
- `buffer` + `intersects` — проверка нормативного отступа

### 4.4 Деплой

Один Dockerfile. CORS настроен на `4dinno.ru`. Данные — GeoJSON-файлы в `/data/`.

## 5. Пресеты объектов

| Slug | Название | Размер | Отступ | Цвет |
|---|---|---|---|---|
| `residential` | Жилое здание | 30x15 м | 10 м | Синий |
| `office` | Офисное здание | 40x20 м | 8 м | Голубой |
| `transformer` | Трансформаторная подстанция | 6x4 м | 15 м | Жёлтый |
| `parking` | Парковка | 25x15 м | 3 м | Серый |
| `warehouse` | Склад | 30x20 м | 5 м | Коричневый |

Единый источник правды — бэкенд (`GET /api/v1/presets`). Extension загружает при старте.

## 6. Данные

Хранение: GeoJSON-файлы в репозитории (`/data/cadastral.geojson`, `/data/protection_zones.geojson`).

Два слоя:
- **Кадастровые участки** — из НСПД, выгрузка по bbox территории ОЭЗ
- **Охранные зоны** — из НСПД (ЗОУИТ) + мок-данные при необходимости

## 7. Что НЕ входит в скоуп хакатона

- Сценарии проектирования (версионирование)
- Импорт DXF/SHP файлов
- Карточка объекта с документами и связями
- RBAC и авторизация
- Интеграция с СМЭВ/ЕГРН/ГИСОГД
- Инжект в сайдбар двойника
- Красные линии, функциональные зоны
- МСК-16 (работаем в WGS-84, как двойник)
- Реестр слоёв / registry-driven design
