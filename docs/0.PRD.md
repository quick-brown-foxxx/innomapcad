# PRD: ГИС-САПР «Иннополис» — Chrome Extension для цифрового двойника
> Версия: 2.0
> Дата: 22.03.2026
> Статус: Draft
> Контекст: хакатон, кейс ОЭЗ «Иннополис»

---

## 1. Резюме продукта

**Проблема.** Проектировщик ОЭЗ «Иннополис» не может проверить, допустимо ли разместить новый объект на территории: кадастровые данные — в Росреестре, охранные зоны — в реестре ЗОУИТ, а цифровой двойник (4dinno.ru/map/territory) не содержит ГИС-слоёв и инструментов валидации.

**Решение.** Chrome Extension (Manifest V3), которое инжектится в существующий цифровой двойник и добавляет:
- ГИС-слои (кадастровые участки, охранные зоны) как deck.gl overlays
- Интерактивное размещение зданий из каталога пресетов
- Мгновенную валидацию ограничений (клиент + сервер)

Два компонента: расширение (фронт) + легковесный FastAPI-бэкенд (валидация, данные).

---

## 2. Целевой пользователь

**Проектировщик** — специалист, размещающий объекты на территории ОЭЗ. Открывает цифровой двойник в Chrome, активирует расширение, видит ГИС-слои поверх 3D-карты, ставит здание, получает вердикт: можно / нельзя и почему.

---

## 3. Демо-сценарий

1. Проектировщик открывает `4dinno.ru/map/territory` в Chrome
2. Расширение автоматически инжектит deck.gl слои: кадастровые участки (полигоны) и охранные зоны (полупрозрачные области)
3. В плавающей панели расширения выбирает пресет «Жилой дом»
4. Кликает на карту — здание появляется как extruded polygon
5. Turf.js мгновенно подсвечивает: объект пересекает охранную зону газопровода (красная обводка)
6. Перемещает здание на свободный участок — Turf.js показывает зелёный статус
7. Нажимает «Проверить» — запрос к FastAPI, Shapely подтверждает: конфликтов нет
8. Результат: JSON-отчёт валидации с координатами и статусом

## 4. Архитектура

```
┌─────────────────────────────────────────────┐
│  Chrome Extension (Manifest V3)             │
│  ┌───────────────┐  ┌────────────────────┐  │
│  │ Content Script │  │ Floating UI Panel  │  │
│  │ deck.gl inject │  │ Пресеты, слои,     │  │
│  │ Turf.js valid. │  │ результат валидации│  │
│  └───────┬───────┘  └────────┬───────────┘  │
└──────────┼───────────────────┼──────────────┘
           │  setProps()       │
           ▼                   │
┌─────────────────┐            │  HTTP (localhost:8000)
│ 4dinno.ru/map   │            ▼
│ deck.gl instance│   ┌────────────────────┐
└─────────────────┘   │ FastAPI + Shapely  │
                      │ GeoJSON in memory  │
                      │ Single container   │
                      └────────────────────┘
```

**Content Script** получает deck.gl instance через React fiber tree (`#deckgl-overlay` → `__reactFiber` → `ref.current.deck`), извлекает конструктор `SolidPolygonLayer` из существующего sublayer, инжектит слои через `deck.setProps()`. MutationObserver переинжектит слои после React re-render.

**Floating UI Panel** — отдельный DOM-элемент расширения (не инжектится в сайдбар двойника). Содержит переключение слоёв, выбор пресета, результат валидации.

## 5. Функциональные требования

| # | Требование | Компонент |
|---|-----------|-----------|
| F1 | Инжекция deck.gl слоя кадастровых участков поверх карты двойника | Extension |
| F2 | Инжекция deck.gl слоя охранных зон поверх карты двойника | Extension |
| F3 | Переключение видимости каждого слоя в UI-панели | Extension |
| F4 | Выбор пресета здания из каталога (5 типов) и размещение кликом по карте | Extension |
| F5 | Мгновенная клиентская валидация через Turf.js: пересечение с зонами, выход за границы участка | Extension |
| F6 | Визуальная обратная связь: зелёная/красная обводка объекта | Extension |
| F7 | Серверная валидация через FastAPI + Shapely по кнопке «Проверить» | Backend |
| F8 | API выдачи GeoJSON-слоёв (участки, зоны) | Backend |
| F9 | API каталога пресетов зданий | Backend |
| F10 | Переинжекция слоёв после React re-render (MutationObserver) | Extension |

## 6. API-контракт

Бэкенд: `http://localhost:8000`

### `GET /health`
```json
{ "status": "ok" }
```

### `GET /api/presets`
```json
[
  { "id": "residential", "name": "Жилой дом", "width": 24, "depth": 14, "height": 9, "floors": 3, "setback_m": 3, "color": "#4A90D9" },
  ...
]
```

### `GET /api/layers/{layer_id}`
`layer_id`: `cadastral_parcels` | `protection_zones`
```json
{ "type": "FeatureCollection", "features": [...] }
```

### `POST /api/validate`
```json
// Request
{ "geometry": { "type": "Polygon", "coordinates": [...] }, "preset_id": "residential" }

// Response
{ "valid": false, "conflicts": [
    { "layer": "protection_zones", "feature_id": "zone_42", "type": "intersection", "description": "Пересечение с охранной зоной газопровода (50 м)", "overlap_area_m2": 12.4 }
  ]
}
```

## 8. Данные

| Слой | Источник | Формат | Описание |
|------|---------|--------|----------|
| Кадастровые участки | НСПД (nspd.gov.ru) | GeoJSON (hardcoded) | Реальные границы участков территории ОЭЗ «Иннополис» |
| Охранные зоны | НСПД (ЗОУИТ) | GeoJSON (hardcoded) | Реальные охранные зоны: газопроводы, ЛЭП, водоохранные |

Данные выгружены заранее, хранятся как статические `.geojson` файлы в бэкенде, загружаются в память при старте. Координаты — WGS-84 (совпадает с системой координат двойника).

## 9. Что НЕ в скоупе

- Авторизация, RBAC, многопользовательский режим
- База данных (PostgreSQL, PostGIS) — всё in-memory
- Сценарии проектирования и версионирование
- Импорт файлов (DXF, SHP)
- Карточка объекта с документами
- Перемещение / поворот объекта (drag-and-drop)
- Интеграция с СМЭВ, ГИСОГД, ЕГРН
- МСК-16 и пересчёт координат — работаем только в WGS-84
- Собственная картографическая подложка (используем двойник)
- Деплой в продакшен, CI/CD
- Красные линии, функциональные зоны, инженерные сети
