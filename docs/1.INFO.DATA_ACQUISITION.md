**US-006 | Research date: 2026-03-23**

Target area: Innopolis, Tatarstan, Russia
- Bbox: lon 48.72-48.76, lat 55.74-55.76
- Center: ~48.74, 55.75

## 1. NSPD (nspd.gov.ru) — Cadastral Data

### What it is

The National Spatial Data Infrastructure portal, replacement for the old PKK (Public Cadastral Map). Contains authoritative cadastral parcel boundaries, buildings, ZOUIT (protection zones), and administrative boundaries from EGRN.

### Test results

| Test | Result |
|---|---|
| WMS GetCapabilities (`/api/aeggis/v2/1/wms?...GetCapabilities`) | **Timeout** (60s). Server unresponsive or blocking. |
| Search API (`/api/geoportal/v2/search/geoportal?query=16:52:070101`) | **Timeout** (60s). Same issue. |

NSPD appears to be inaccessible from outside Russia or heavily rate-limited. The existing research in `1.INFO.DATA_SOURCES.md` notes that access from foreign IPs is blocked. On a hackathon inside Russia, this should work.

### Indirect access via rosreestr2coord

The `rosreestr2coord` Python library (v5.3.3, released 2026-02-04, actively maintained) wraps the NSPD API:

- **Install:** `pip install rosreestr2coord`
- **Usage:** `rosreestr2coord -c 16:52:070101:123 -w -a` (get parcel as WGS84 GeoJSON with attributes)
- **Batch:** `rosreestr2coord -l cadastral_numbers.txt -w -a`
- **Programmatic:**
  ```python
  from rosreestr2coord import Area
  area = Area("16:52:070101:123")
  geojson = area.to_geojson()
  ```
- **Area types supported:** land plots (1), buildings (5), cadastral quarters (2), districts (3), ZOUIT/zones (5), territorial zones (7)
- **Limitation:** Search is by individual cadastral number only. **No bbox/area search.** To get all parcels in an area, you need a list of cadastral numbers first.
- **Warning:** Documentation notes risk of IP blocking during batch processing.

### Assessment

- **Format:** GeoJSON (via rosreestr2coord) or WMS tiles (direct)
- **Data quality:** Authoritative EGRN data; coordinates are approximate (not equivalent to official cadastral extract)
- **Auth:** None for public data; ESIA (Gosuslugi) for extended features
- **Accessibility:** Blocked outside Russia. Inside Russia: works, but no bulk bbox download. Need to know specific cadastral numbers.
- **Verdict:** Usable on hackathon inside Russia for individual parcels. Not practical for bulk area download without a pre-existing list of cadastral numbers.

## 2. OpenStreetMap Overpass API — Best Available Bulk Source

### Test results

Tested with real Overpass queries against the Innopolis bbox `(55.74, 48.72, 55.76, 48.76)`:

| Query | Features Found | Quality |
|---|---|---|
| Buildings (`way["building"]`) | **~210 ways** | Moderately rich. Apartments (6-9 floors, flat counts), university buildings, dormitories, sports center, clinic, hotel, data center, substations. Many have `building:levels`, `start_date`, `energy_class`. Some are just `building=yes`. |
| Landuse (`way["landuse"]`) | **~60+ ways** | Residential, recreation_ground, grass, greenfield, industrial, construction, forest, education. Good zoning coverage for a planned city. |
| Leisure | **~60+ features** | Parks, playgrounds (15+), sports pitches (13+), stadium, dog park, track. |
| Amenity | **~90+ features** | Parking (60+), kindergartens, schools, university, clinic, cafes, community center. |
| Boundary (relations) | Administrative boundary of Innopolis settlement exists. |

### Example query (buildings as GeoJSON)

```
https://overpass-api.de/api/interpreter?data=
[out:json][timeout:25];
(way["building"](55.74,48.72,55.76,48.76););
out body;>;out skel qt;
```

Convert to GeoJSON with `osmtogeojson` (npm package) or process in Python with `osm2geojson`.

### Assessment

- **Format:** JSON (Overpass native) -> convertible to GeoJSON
- **Data quality:** Good for a young planned city. ~210 buildings with moderate attributes. Landuse zoning is comprehensive. Not cadastral-grade (no cadastral numbers, no legal boundaries), but geometrically useful.
- **Auth:** None required
- **Rate limits:** Generous for reasonable use. No API key needed.
- **License:** ODbL (free use with attribution: "(c) OpenStreetMap contributors")
- **Verdict:** Best source for bulk area download. Use as base layer for buildings, roads, landuse. Supplement with cadastral data from NSPD when available.

## 3. 4dinno.ru — Innopolis Digital Twin

### What it is

Official digital twin platform for Innopolis city. Claims 250+ information layers, 110,000+ digitized objects, 500+ km of city networks. Built for municipal authorities.

### Test results

| Test | Result |
|---|---|
| Category API (`/api/v2/layers/ru/layers/anonymous/`) | Returns **4 empty categories**: Buildings, Transport, City Improvement, Territorial Planning. All have `layers: []` or `layers: null`. No actual data in anonymous/public mode. |
| GeoJSON files (`/media/layers/buildings.geojson`) | **502 Bad Gateway.** File does not exist or is not served. |
| Web search for API docs | **No public documentation found.** The platform is not designed for external API consumers. |

### Assessment

- **Format:** Unknown (API returns category taxonomy only, no actual data)
- **Data quality:** Platform claims rich data, but none is publicly accessible
- **Auth:** Registration required for actual data access. Anonymous API returns empty shells.
- **Verdict:** Not usable without credentials. Would need to contact Innopolis city administration (city@innopolis.ru) for API access. Not viable for hackathon unless organizers provide credentials.

## 4. rosreestr2coord Library — Detailed Evaluation

### Current status (as of 2026-03-23)

- **Version:** 5.3.3 (2026-02-04)
- **Maintenance:** Active, healthy (releases every few months)
- **API backend:** NSPD (migrated from PKK in v5.0.0, December 2024)
- **PyPI:** https://pypi.org/project/rosreestr2coord/

### Capabilities

| Feature | Supported |
|---|---|
| Get parcel geometry by cadastral number | Yes |
| Get building geometry by cadastral number | Yes |
| Get ZOUIT (protection zones) | Yes (type 5) |
| Get territorial zones | Yes (type 7) |
| Export to GeoJSON | Yes |
| Export to KML | Yes |
| WGS84 conversion | Yes (`-w` flag) |
| Add attributes (area, category, VRI) | Yes (`-a` flag) |
| Batch processing | Yes (from file list) |
| **Bbox/area search** | **No** |
| Proxy support | Yes |
| Request caching | Yes |

### Key limitation

No way to discover cadastral numbers within a bbox. The tool requires you to already know the cadastral number. For Innopolis, you would need to manually find numbers via NSPD web interface first.

### MCP server

There is an MCP server wrapper: `mcp-rosreestr` (github.com/yasg1988/mcp-rosreestr) that exposes rosreestr2coord through MCP protocol. Could be interesting for AI-assisted workflows.

## 5. Summary and Recommendation

### Source comparison

| Source | Bulk Area Download | Data Quality | Accessibility | Auth |
|---|---|---|---|---|
| OSM Overpass | Yes (bbox) | Good (not cadastral) | Excellent | None |
| NSPD (direct) | No (WMS tiles only) | Authoritative | Blocked outside RU | None/ESIA |
| rosreestr2coord | No (by cad. number) | Authoritative | Needs known cad. numbers | None |
| 4dinno.ru | No | Unknown | Requires registration | Credentials |

### Recommended approach for hackathon

**Strategy: OSM base + generated cadastral mock data**

1. **Base layer from OSM** (real data):
   - Fetch buildings, landuse, roads, amenities via Overpass API
   - Convert to GeoJSON with `osm2geojson`
   - This gives real geometry for ~210 buildings and comprehensive zoning

2. **Cadastral parcels: generate realistic mock data with Shapely** (fallback):
   - Innopolis is a planned city with a grid-like layout — parcels follow a regular pattern
   - Generate rectangular parcels on a grid centered at (48.74, 55.75)
   - Assign fake but realistic cadastral numbers in format `16:52:XXXXXX:NNN`
   - Add attributes: area (sq.m), category (land for buildings), VRI (for IT park / residential / education)

3. **Protection zones: generate realistic mock data** (fallback):
   - Linear zones (gas pipeline, power line) crossing the territory diagonally
   - Buffer zones around utility corridors using Shapely `buffer()`
   - Reference real normative widths (gas: 150m each side per PP RF 578, power line: depends on voltage per PP RF 160)

4. **If on hackathon inside Russia:**
   - Try NSPD WMS tiles as visual layer (add Referer header)
   - Use rosreestr2coord for a few known parcels to demonstrate real data integration
   - Show the system can accept both real and mock cadastral data

### Why not just use real NSPD data?

- No bbox search means we cannot bulk-download all parcels in an area
- NSPD is blocked outside Russia (development/testing constraint)
- Even inside Russia, batch scraping risks IP blocking
- For a hackathon demo, a mix of real OSM + realistic mock cadastral data is more reliable than depending on government API availability

### Mock data generation approach (for `generate_mock_data.py`)

```python
# Pseudocode for generating Innopolis mock cadastral data
from shapely.geometry import box, LineString
import json

CENTER = (48.74, 55.75)  # Innopolis center
GRID_SIZE = 20  # parcels
PARCEL_WIDTH = 0.001  # ~80m in degrees at this latitude

# Generate grid of cadastral parcels
parcels = []
for i in range(GRID_SIZE):
    for j in range(GRID_SIZE):
        lon = CENTER[0] - 0.01 + i * PARCEL_WIDTH
        lat = CENTER[1] - 0.01 + j * PARCEL_WIDTH
        parcel = box(lon, lat, lon + PARCEL_WIDTH * 0.9, lat + PARCEL_WIDTH * 0.9)
        parcels.append({
            "type": "Feature",
            "properties": {
                "cadastral_number": f"16:52:070101:{100 + i*GRID_SIZE + j}",
                "area_sqm": round(parcel.area * 1e10, 1),  # approximate
                "category": "Земли населённых пунктов",
                "vri": ["ИЖС", "Многоквартирный дом", "Образование", "ИТ-парк"][hash((i,j)) % 4]
            },
            "geometry": parcel.__geo_interface__
        })

# Generate protection zone (gas pipeline crossing diagonally)
pipeline = LineString([(48.72, 55.742), (48.76, 55.758)])
protection_zone = pipeline.buffer(0.0015)  # ~120m buffer
```

This approach produces a realistic dataset that demonstrates the system's ability to handle cadastral + protection zone intersection analysis.
