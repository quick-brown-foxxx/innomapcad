# Данные и их источники — ОЭЗ «Иннополис»

## Краткая сводка

| Источник | Доступ | Авторизация | Формат данных | Юр. статус |
|---|---|---|---|---|
| НСПД (бывш. ПКК) | REST API + WMS | Без ключей (публичная часть); ЕСИА (расширенная) | JSON, PNG-тайлы | Общедоступные сведения ЕГРН, 218-ФЗ ст. 7 п. 5 |
| Росреестр ЕГРН (полные выписки) | СМЭВ / Госуслуги | ЭЦП юрлица / ЕСИА | XML (схемы утверждены приказами) | Платная услуга, часть данных ДСП |
| OpenStreetMap | Overpass API | Без ключей | GeoJSON, XML | ODbL — свободное использование с указанием источника |
| Открытые данные Татарстана | REST API (CKAN) | Без ключей | CSV, JSON, XML | Свободное использование, бессрочно, безвозмездно |
| ГИСОГД Татарстана | Закрытый портал | Служебный доступ | Не документирован публично | Ведомственная система |
| Геопортал РТ | Веб (демо-доступ) | Логин/пароль | WMS-слои | Ведомственная система |
| Проектная документация | Ручной импорт | Нет API | DXF, DWG, IFC, PDF | Собственность застройщика/заказчика |
| Техпланы / техпаспорта | СМЭВ или ручной импорт | ЭЦП / нет | XML (стандарт Росреестра) | Часть ЕГРН |
| ППТ | Ручной импорт | Нет API | DXF, PDF | Утверждаются муниципалитетами |

## 1. НСПД — главный автоматический источник

### Что произошло

С 23 декабря 2024 года Публичная кадастровая карта (pkk.rosreestr.ru) перенесена на платформу НСПД — nspd.gov.ru. Старый домен редиректит на новый. Это принципиально важно: **старые эндпоинты ПКК могут не работать**, нужно ориентироваться на API НСПД.

### Что доступно

НСПД предоставляет WMS-сервис для тайлов. Формат URL (из реальных конфигов SAS.Планета):

```
https://nspd.gov.ru/api/aeggis/v2/{layer_id}/wms?SERVICE=WMS&VERSION=1.3.0&REQUEST=GetMap
  &FORMAT=image/png&TRANSPARENT=true&LAYERS={layer_id}
  &CRS=EPSG:3857&BBOX={bbox}&WIDTH=256&HEIGHT=256
```

Известные ID слоёв: кадастровые участки, кадастровые кварталы, здания, ЗОУИТ. Требуется заголовок `Referer: https://nspd.gov.ru`.

Также есть REST API для поиска объектов по кадастровому номеру — существует готовая Python-библиотека `rosreestr2coord` и даже MCP-сервер (github.com/yasg1988/mcp-rosreestr), которые обёртывают API НСПД.

### Какие данные отдаёт

- Кадастровые участки: границы (геометрия), кадастровый номер, площадь, категория земель, ВРИ, кадастровая стоимость, форма собственности.
- Здания / сооружения / ОНС: контуры, этажность, материал стен, год постройки, назначение.
- ЗОУИТ (зоны с особыми условиями): охранные зоны инженерных сетей, водоохранные зоны, зоны объектов культурного наследия. Это критично для задачи размещения объектов.
- Границы: административные, муниципальные, населённых пунктов, территориальные зоны.

### Юридический статус

Сведения ЕГРН на НСПД являются общедоступными по 218-ФЗ «О государственной регистрации недвижимости», статья 7, пункт 5. Ранее на ПКК были условия «только некоммерческое использование» — они были убраны как не соответствующие закону. Сведения носят **справочный характер** и не являются юридическим документом.

**Важный нюанс**: спутниковые снимки и картографическая основа НСПД (подложка) **не являются** общедоступными сведениями ЕГРН и не попадают под 218-ФЗ. Использовать можно только векторные данные из реестра.

### Ограничения

- Доступ по IP из-за рубежа заблокирован (защита от DDoS). На хакатоне в России — не проблема.
- `authkey` в URL периодически меняется. Для хакатона: `authkey=0` работает.
- Полный функционал НСПД требует авторизации через ЕСИА (Госуслуги).
- Данные обновляются ежедневно, но фактическая задержка зависит от региона.
- НСПД интегрирована со 120+ информационными системами и работает в 64 регионах (на конец 2025).

## 2. Росреестр — полные данные через СМЭВ

### Что это

СМЭВ (Система межведомственного электронного взаимодействия) — это способ получения полных выписок ЕГРН программно. В отличие от НСПД (справочные данные), через СМЭВ приходят юридически значимые документы.

### Что даёт сверх НСПД

- Полные техпланы с поэтажными планами.
- Информация о правообладателях (ограничена с марта 2023 — 266-ФЗ, данные о физлицах-собственниках доступны только с их согласия).
- Обременения, аресты, залоги.
- История переходов прав.

### Доступ

Требуется регистрация юрлица в СМЭВ, ЭЦП, оплата за выписки. ОЭЗ «Иннополис» как государственная структура вероятно имеет доступ или может получить. **На хакатоне — мокируем**: кладём пример XML-выписки, показываем парсинг.

### Формат

XML-схемы техпланов и межевых планов стандартизированы приказами Росреестра. Схемы опубликованы, парсер реализуем.

## 3. Открытые данные Татарстана

### Портал opendata.tatar.ru

REST API на базе CKAN. 700+ наборов данных. Доступ свободный, бессрочный, безвозмездный, без договора (прямо написано на портале).

### Портал data.kzn.ru (Казань)

Открытые данные города Казань. Может содержать данные об инфраструктуре, транспорте, социальных объектах — полезно как дополнительный слой.

### Что из этого полезно

Прямых кадастровых или градостроительных данных там нет — это в основном статистика, бюджет, реестры учреждений. Но адреса и реестры объектов могут пригодиться для обогащения семантики.

## 4. ГИСОГД Татарстана

### Что это

Государственная информационная система обеспечения градостроительной деятельности. Ведёт ГБУ «Институт пространственного планирования РТ». Содержит ППТ, ПЗЗ, генпланы, красные линии — именно те данные, которые критичны для задачи размещения объектов.

### Доступ

Публичного API нет. Портал закрытый, для служебного использования органами власти. Теоретически НСПД должна интегрироваться с ГИСОГД (это одна из задач госпрограммы), но по Татарстану статус интеграции неясен.

### Что делать на хакатоне

Мокировать. Берём реальный DXF-файл ППТ Иннополиса (если удастся получить от организаторов) или генерируем тестовые данные красных линий и функциональных зон. Показываем, что система принимает и отображает.

## 5. Геопортал Республики Татарстан

### URL

geoportal.glonass-112.net — демо-доступ: логин `demo`, пароль `demo`.

### Что это

WMS-слои с данными о территории РТ. Создан в рамках проекта «ГЛОНАСС+112». Может содержать слои инженерных сетей, социальных объектов, транспортной инфраструктуры.

### Практическое значение

Стоит проверить на хакатоне, какие слои доступны по WMS — если есть что-то полезное (инженерные сети, дороги), можно подключить напрямую как внешний WMS-источник в нашу систему. Это бесплатно и не требует импорта.

## 6. OpenStreetMap

### Доступ

Overpass API — без ключей, без лимитов (при разумном использовании). Данные экспортируются в GeoJSON.

### Что полезно

Дороги, здания (контуры), POI (школы, магазины, инфраструктура), landuse (зонирование территории). По Иннополису покрытие может быть неполным — это молодая территория.

### Юридический статус

Лицензия ODbL: можно использовать свободно, в том числе коммерчески, при условии указания источника «© OpenStreetMap contributors» и share-alike (производные данные — под той же лицензией). Для внутренней системы ОЭЗ это не проблема.

## 7. Проектная документация (DXF/DWG/IFC)

### Откуда берётся

От застройщиков, проектных организаций, подрядчиков. Передаётся на цифровых носителях или через облако. Единого хранилища нет.

### Форматы

- **DXF** (Drawing Exchange Format) — открытый текстовый формат Autodesk. Читается GDAL/OGR (`ogr2ogr`). Основной формат для 2D-планов.
- **DWG** — бинарный проприетарный формат Autodesk. GDAL читает через библиотеку LibreDWG (или ODA SDK). Поддержка хуже, чем DXF.
- **IFC** (Industry Foundation Classes) — открытый стандарт BIM (ISO 16739). Для 3D-моделей зданий. Библиотека IfcOpenShell (Python). На хакатоне — скорее на будущее.
- **PDF** — чертежи в растре. Автоматическая обработка требует OCR/векторизации. На хакатоне — не реализуемо.

### Юридический статус

Проектная документация — собственность заказчика/застройщика. Для использования в системе ОЭЗ нужно правовое основание (договор, регламент). На хакатоне — используем тестовые данные.

### Проблемы при импорте

- **Система координат**: DXF-файлы из AutoCAD часто в условной (локальной) СК. Привязка к МСК-16 или WGS-84 требует знания точки привязки.
- **Структура слоёв**: каждый проектировщик структурирует слои по-своему. Маппинг слоёв DXF → слои системы — полуавтоматический.
- **Качество геометрии**: незамкнутые полигоны, дублирующиеся вершины, самопересечения. PostGIS `ST_MakeValid()` решает часть проблем.

## 8. Система координат МСК-16

### Что это

Местная система координат Республики Татарстан. Используется для кадастровых работ и ведения ЕГРН на территории региона. Основана на проекции Гаусса-Крюгера, эллипсоид Красовского.

### Параметры

МСК-16 разбита на три 3-градусные зоны. Параметры для PROJ.4 / QGIS / MapInfo **опубликованы** и общедоступны (terraingis.ru, mapbasic.ru, geoproj.ru, geobridge.ru):

```
МСК-16 зона 1: центральный меридиан 49°02', false easting 1300000, false northing -5709414.70
МСК-16 зона 2: центральный меридиан 52°02', false easting 2300000, false northing -5709414.70
МСК-16 зона 3: центральный меридиан 55°02', false easting 3300000, false northing -5709414.70
```

### Юридический нюанс

Формально параметры МСК имеют гриф ДСП (для служебного пользования). На практике параметры МСК-16 опубликованы на десятках профессиональных геодезических ресурсов, включены в ГОСТ 51794-2008, и используются в коммерческих ГИС (Панорама, MapInfo). Для системы ОЭЗ (государственная структура) использование МСК-16 не просто допустимо — это **требование**: кейс прямо указывает на необходимость работы в МСК-16.

### Техническая реализация

Регистрация как кастомный SRID в PostGIS `spatial_ref_sys` — один SQL-insert при инициализации. Пересчёт: `ST_Transform` в базе, `pyproj` в API. Инструменты для проверки: geoproj.ru, geobridge.ru — онлайн-калькуляторы.

Иннополис расположен примерно на 48.75°E — это **зона 1** МСК-16.

## 9. Охранные зоны (ЗОУИТ)

### Источник

С 2022 года Росреестр ведёт реестр ЗОУИТ. Данные доступны через НСПД как отдельный слой. Типы зон: охранные зоны газопроводов, ЛЭП, водопроводов, зоны санитарной охраны, водоохранные зоны, зоны объектов культурного наследия.

### Полнота

Реестр ЗОУИТ заполняется постепенно. Не все зоны внесены — особенно для строящихся объектов (что как раз актуально для Иннополиса). Система должна поддерживать **ручное добавление** охранных зон сверх того, что есть в реестре.

### Нормативная база

Размеры охранных зон определяются федеральными НПА:
- Газопроводы: Правила охраны магистральных трубопроводов (ПП РФ №578).
- ЛЭП: ПП РФ №160.
- Водоохранные зоны: Водный кодекс, ст. 65.

Эти нормативы — основа для правил валидации в системе (расстояния, запреты на строительство).

## 10. Практические рекомендации для хакатона

### Что подключать реально

1. **НСПД** (WMS + поиск по кад. номеру) — реальные кадастровые данные Иннополиса.
2. **OSM** через Overpass — дороги, здания, базовая инфраструктура.
3. **Тестовый DXF** — подготовить 1–2 файла с геометрией проектируемых объектов.

### Что мокировать

1. **СМЭВ/ЕГРН** — положить XML-файл техплана, показать парсинг.
2. **ГИСОГД** — создать тестовый слой красных линий в PostGIS.
3. **Охранные зоны строящихся объектов** — создать вручную, показать валидацию.
